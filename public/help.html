2: CSS modifications

Most of my CSS is as jss inside of each component. This gets compiled down to classes named jss# to avoid collisions.

I didn't use any css templates, but I did include a theme modified from react-theming.github.io/create-mui-theme. It is less of a css template and more colors and sizes and whatnot in plain json. The generated file is in src/theme/teal.

I did all the styling for the navbar and text manually in public/appbar.css just to be safe

3: Use

The file manager is the first icon, and has the upload and download functionality. Upload immediately uploads your file and adds it to the list of available files.

The save dialog is the floppy disc icon second from the left. Files are stored flat in the same folder. After saving a file is available to download or reopen from the file manager.

Undo and redo for coloring is based on strokes. Undoing will move back to the point at which the current click began.

Bugs

I can't figure out why the color input calls it's onChange every time the color gets changed. I was under the impression that onChange should only happen when select is hit and onInput was for all changes, but they seem to both be the same on my machine on ff and chrome. This means that hitting undo will undo a single step of the color change, and if you drag the cursor around each step will add a new color to be undone, which adds up quickly. I recommend either clicking carefully or just entering in a value. I think this still fits within the assignment spec, but it's annoying and caught me off guard.

It doesn't scale that well in super low height windows, but I don't think the window is useable at that scale either way. Can't really solve it without measuring things in js, and I don't want to do that.
